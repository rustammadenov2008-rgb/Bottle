# Debate Arena — спор Gemini и Grok

Веб-приложение, где Gemini и Grok спорят друг с другом на заданную тему,
а табло считает слова и признаки согласия каждой стороны.

Стек: Next.js (App Router) на Vercel, Neon Postgres (через Vercel Storage)
для хранения споров, Gemini API + xAI API на сервере (ключи не светятся
в браузере).

## Как это работает

1. На главной странице задаёшь тему и "личность" (system prompt) для
   каждого бота, выбираешь число раундов.
2. Создаётся запись спора в базе, открывается страница `/debate/{id}`.
3. Боты по очереди отвечают друг другу — каждый ход дёргает соответствующий
   API (Gemini или xAI/Grok) на сервере, ответ сохраняется в базу.
4. Ссылку на спор можно скопировать и отправить кому угодно — страница
   читает спор из базы по id, ключи API при этом никуда не передаются.
5. Если ключ xAI не задан, на балансе нет денег или запрос упал — приложение
   переходит в ручной режим и просит вставить ответ Grok вручную (например,
   скопировав его с grok.com), чтобы спор не подвисал.

## Локальный запуск

```bash
npm install
cp .env.example .env.local
# заполни .env.local своими ключами и строкой подключения к базе
npm run db:migrate   # создаёт таблицу debates в базе данных
npm run dev
```

Открой http://localhost:3000.

## Деплой на Vercel

### 1. Создай базу данных

В панели Vercel: **Storage → Create Database → Neon Postgres** (это
нативная интеграция Vercel с Neon — полноценный managed Postgres).
После создания Vercel сам пропишет переменную `DATABASE_URL` (или
`POSTGRES_URL`, обе поддерживаются) в Environment Variables проекта.

### 2. Примени миграцию

Скопируй строку подключения из вкладки **.env.local** в разделе базы данных
и выполни локально один раз:

```bash
DATABASE_URL="<строка из Vercel>" npm run db:migrate
```

Это применит оба файла из `migrations/` по порядку — `001` создаёт таблицу,
`002` безопасно пересоздаёт её под актуальную схему (если до этого что-то
уже было накатано со старой версией проекта).

Либо выполни содержимое файлов через SQL-редактор Neon
(Vercel Storage → Open in Neon → SQL Editor).

### 3. Задай переменные окружения проекта

В **Project Settings → Environment Variables** добавь:

- `GEMINI_API_KEY` — ключ с https://aistudio.google.com/apikey
- `XAI_API_KEY` — ключ с https://console.x.ai (нужно привязать карту,
  бесплатного тира у xAI нет)
- `GEMINI_MODEL` (опционально) — по умолчанию `gemini-2.5-flash`
- `GROK_MODEL` (опционально) — по умолчанию `grok-4.3`

`DATABASE_URL` уже будет задан автоматически после создания базы на шаге 1.

### 4. Задеплой

```bash
npx vercel --prod
```

или подключи репозиторий на vercel.com и задеплой через UI.

## Структура проекта

```
src/
  app/
    page.tsx                   главная страница — форма создания спора
    debate/[id]/page.tsx       страница просмотра/шаринга конкретного спора
    api/debate/start           создание нового спора
    api/debate/[id]            получение спора по id
    api/debate/[id]/turn       выполнение очередного хода (Gemini или Grok)
    api/debate/[id]/manual-turn  ручной ввод ответа Grok (fallback)
  components/                  UI: карточка боя, реплики, табло
  lib/
    db.ts                      подключение к Neon
    gemini.ts / grok.ts        клиенты моделей
    debate-engine.ts           построение промптов, подсчёт реплик
    analytics.ts                подсчёт слов и признаков согласия
migrations/                    SQL миграции
scripts/migrate.ts             запуск миграций (npm run db:migrate)
```

## Известные ограничения

- Аналитика "признаков согласия" — это простая эвристика по ключевым словам
  в тексте реплики, а не полноценный семантический анализ.
- Автопрогон ходов держится на стороне браузера (клиент сам дёргает
  следующий ход после получения предыдущего), поэтому страница спора должна
  быть открыта, чтобы спор продолжался.
- Имена моделей (`GEMINI_MODEL`, `GROK_MODEL`) заданы через переменные
  окружения, а не зашиты в код — обе компании периодически переименовывают
  и заменяют модели, так что при устаревании имени достаточно поменять
  переменную, без правок кода.
